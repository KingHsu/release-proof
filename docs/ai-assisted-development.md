# AI-assisted development disclosure

This project was designed and implemented with AI coding assistance. That is part of the subject matter: generated code still needs explicit acceptance criteria and review evidence.

Human review responsibilities remain:

- confirm product scope and non-goals;
- inspect filesystem, subprocess, MCP, and secret-handling boundaries;
- execute offline tests and reproduce failures;
- validate dependency versions and generated lock files;
- review evaluation labels and avoid publishing fixture metrics as production results;
- run any DeepSeek or GitHub integration manually with minimum permissions and a defined budget.

AI output is not treated as evidence that another AI-generated implementation is correct. Tests, parsed reports, deterministic invariants, and traceable source references form the review evidence.
